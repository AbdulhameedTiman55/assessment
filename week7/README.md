## Learning Objectives
The activities in this folder demonstrate:
- Evaluation of defect report quality (Lab 7A)
- Creation of formal defect reports (Lab 7B)
- Use of GitHub Issues for defect tracking
- Evidence-based defect reporting with screenshots

---

## Defect Reporting Tool
All defects were logged using **GitHub Issues**, which provides:
- Unique defect identifiers
- Status tracking (Open / Closed)
- Clear traceability between defects and system behaviour
- Support for screenshots and additional evidence

Each issue includes:
- A clear and descriptive title
- Steps to reproduce
- Expected vs actual behaviour
- Severity and priority classification
